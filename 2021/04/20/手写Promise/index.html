     Promise相信大家都不陌生，对于其实现相信大家都有一定的了解。下面就简单的复习一下promise的实现：


一.核心逻辑实现



  • 先看一个原生的例子，再来实现它

const promise  = new Promise((resolve, reject) => {
    resolv(‘success’) // 或者reject(‘’fail)
})
promise.then(value => {
    console.log(‘resolve’, value)
}, value => {
     console.log(‘reject’, value)
})


  • 基本原理

❝

1.Promise 是一个类，在执行这个类的时候会传入一个执行器，这个执行器会立即执行
2.Promise 会有三种状态(Pending等待 Fulfiled完成 Reject 失败)
3.状态只能有Pending->Fulfiled 或者 Pending -> Reject。状态一旦修改不会发生变化
4.Promise 中使用 resolve 和 reject 两个函数来更改状态
5.then 方法内部做但事情就是状态判断，如果状态是成功，调用成功回调函数，如果状态是失败，调用失败回调函数


❞

1.新建myProise类，传入执行器


class MyPromise {
  constructor(executer) {
    executer() //executor 是一个执行器，进入会立即执行
  }
}

2.传入resolve和reject


class MyPromise {
  constructor(executer) {
    executer(this.resolve, this.reject) //executor 是一个执行器，进入会立即执行
  }
  resolve = () => {}
  reject = () => {}
}

3.状态和结果管理


const PENDING = ‘pending’
const FULFILLED = ‘fulfilled’
const REJECTED = ‘rejected’
class MyPromise {
  constructor(executer) {
    executer(this.resolve, this.reject) //executor 是一个执行器，进入会立即执行
  }
  status = PENDING;//当前状态
  value = null;// 成功的值
  reason = null;// 失败的值
  resolve = (val) => {
    if (this.status === PENDING) {
      this.status = FULFILLED
      this.value = val
    }
  }
  reject = (val) => {
    if (this.status === PENDING) {
      this.status = REJECTED
      this.reason = val
    }
  }
}

4.then的简单实现


then = (onFulfiled, onReject) => {
    if (this.status === FULFILLED) {
      onFulfiled(this.value)
    } else if (this.status === REJECTED) {
      onReject(this.reason)
    }
  }

  • 现在异步的问题

❝

主线程代码立即执行，setTimeout 是异步代码，then 会马上执行，这个时候判断 Promise 状态，状态是 Pending，然而之前并没有判断等待这个状态。下面就继续改造：


❞

1.缓存失败成功回调


fulfilCallback = null; // 成功的回调
rejectCallback = null; // 失败的回调
then = (onFulfiled, onReject) => {
    if (this.status === FULFILLED) {
      onFulfiled(this.value)
    } else if (this.status === REJECTED) {
      onReject(this.reason)
    } else if (this.status === PENDING) {
      this.fulfilCallback = onFulfiled
      this.rejectCallback = onReject
    }
  }

2.resolve和reject中调用回调


resolve = (val) => {
    if (this.status === PENDING) {
      this.status = FULFILLED
      this.value = val
      this.fulfilCallback && this.fulfilCallback(val)
    }
  }
  reject = (val) => {
    if (this.status === PENDING) {
      this.status = REJECTED
      this.reason = val
      this.rejectCallback && this.rejectCallback(val)
    }
  }

三.实现then多次调用


❝

Promise 的 then 方法是可以被多次调用的。这里如果有三个 then 的调用，如果是同步回调，那么直接返回当前的值就行；如果是异步回调，那么保存的成功失败的回调，需要用不同的值保存，因为都互不相同。之前的代码需要改进。


❞

1.回调变量改为数组

rejectCallback = []; // 失败的回调

then = (onFulfiled, onReject) => {
  if (this.status === FULFILLED) {
    onFulfiled(this.value)

    onReject(this.reason)

2.循环调用回调


resolve = (val) => {
    if (this.status === PENDING) {
      this.status = FULFILLED
      this.value = val
      while (this.fulfilCallback.length) {
        this.fulfilCallback.shift()(val)
      }
    }
  }
  reject = (val) => {
    if (this.status === PENDING) {
      this.status = REJECTED
      this.reason = val
      while (this.rejectCallback.length) {
        this.rejectCallback.shift()(val)
      }
    }
  }

四.实现then链式调用


❝

then 方法要链式调用那么就需要返回一个 Promise 对象then 方法里面 return 一个返回值作为下一个 then 方法的参数，如果是 return 一个 Promise 对象，那么就需要判断它的状态


❞


  • 当前链式调用的问题

  setTimeout(() => {
    resolve(‘success’)
  }, 2000);
})
function otherPromise() {
  return new MyPromise((resolve, reject) => {
    resolve(‘other’)
  })
}
promise.then(value => {
  console.log(‘resolve’, value)
  return  otherPromise()
}).then(val => {
  console.log(‘resolve’, val)
})

// 现在会报错 Cannot read property ’then’ of undefined


  • 继续修改then方法

then = (onFulfiled, onReject) => {
    let promise2 = new MyPromise((resolve, reject) => {
      if (this.status === FULFILLED) {
        const x = onFulfiled(this.value);
        resolvePromise(x, resolve, reject);
      } else if (this.status === REJECTED) {
        onReject(this.reason)
      } else if (this.status === PENDING) {
        this.fulfilCallback.push(onFulfiled)
        this.rejectCallback.push(onReject)
    }
  })
  return promise2
}

function resolvePromise(x, resolve, reject) {
  if (x instanceof MyPromise) {
      x.then(resolve, reject)
  } else {
    resolve(x)
  }
}

五.then链式调用识别是否返回本身


❝

如果 then 方法返回的是自己的 Promise 对象，则会发生循环调用，这个时候程序会报错


❞



  • 直接用我们上面的代码执行，会报如下错误


❝

从错误提示可以看出，我们必须要等 promise2 完成初始化。这个时候我们就要用上宏微任务和事件循环的知识了，这里就需要创建一个异步函数去等待 promise2 完成初始化，前面我们已经确认了创建微任务的技术方案 –> queueMicrotask, 下面继续改造代码：


❞

then = (onFulfiled, onReject) => {
    let promise2 = new MyPromise((resolve, reject) => {
    if (this.status === FULFILLED) {
      queueMicrotask(() => {
        // 获取成功回调函数的执行结果
        const x = onFulfiled(this.value);
        // 传入 resolvePromise 集中处理
        resolvePromise(promise2, x, resolve, reject);
      }) 
    } else if (this.status === REJECTED) {
      onReject(this.reason)
    } else if (this.status === PENDING) {
      this.fulfilCallback.push(onFulfiled)
      this.rejectCallback.push(onReject)
    }
  })
  return promise2
}

function resolvePromise(promise2, x, resolve, reject) {
  if (promise2 === x) {
    return reject(new TypeError(‘Chaining cycle detected for promise #<Promise>’))
  }
  if (x instanceof MyPromise) {
    x.then(resolve, reject)
  } else {
    resolve(x)
  }
}
