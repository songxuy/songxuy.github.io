GitHub Actions 自动部署

四月 28, 2021
  • GitHub Action
  • 自动部署

对于我们部署在github上的项目来说，每次更新都需要重新打包、上传、在通过路径拼接访问到项目。使用github actions之后我们仅仅需要将修改的代码提交，其他的工作交给github actions。


什么是 GitHub Actions


❝

GitHub Actions 是 GitHub 的持续集成服务，持续集成由很多操作组成，比如抓取代码、运行测试、登录远程服务器、发布到第三方服务等等，GitHub 把这些操作称为 actions。


❞


配置 GItHub Actions


新建github仓库


❝

也可以直接使用现有的仓库， 新建过程就不在赘述了


❞

其中我们使用：



  • main分支 存储项目源代码
  • gh-pages分支 存储打包之后的代码

❝

gh-pages 分支，是 GitHub Pages 服务的固定分支，可以通过 HTTP 的方式访问到这个分支的静态文件资源。也就是我们直接接访问你的github主域名/ 项目名 就能访问到了。


❞

创建GitHub Token



  • 主页->settings->Developer settings -> personal access tokens -> generate new token -> 勾选repo 和 workflow




  • 注意：新生成的 Token 只会显示一次，保存起来，后面要用到。如有遗失，重新生成即可。

仓库中添加secret



  • 将新创建的token添加到项目仓库的secret中。

Settings -> Secrets -> new repository secret



❝

需要注意的是设置的名字需要记住，后面会用到


❞

创建 Actions 配置文件



  • 在项目根目录下创建.github文件夹
  • 在.github文件夹中创建 workflows文件夹（来存储workflow配置文件）
  • 在workflows中新建deploy.yml

❝

GitHub 只要发现.github/workflows目录里面有.yml文件，就会自动运行该文件。


❞


deploy.yml 内容


name: deploy
on:
  push:
    branches: [main] # main 分支有 push 时触发
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Setup Node.js v12.x
        uses: actions/setup-node@v1
        with:
          node-version: ‘12.x’
      - name: Install
        run: yarn install # 安装依赖
      - name: Build
        run: CI=false npm run build # 打包
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3 # 使用部署到 GitHub pages 的 action
        with:
          publish_dir: ./build # 部署打包后的 dist 目录
          github_token: $ # 你自己的secret 名
          user_name: 
          user_email: *
          commit_message: Update jj-react # 部署时的 git 提交信息，自由填写

❝

具体的配置信息可以参考后面的文章链接


❞

验证结果


❝

当我们push到main分支时， 可以点击actions查看运行情况，以及结果历史等。
