组件导入

十二月 24, 2022
  • JS
❝

为了一些组件通过脚本解析导入，需要在组件开发中增加一些配置和描述。


下面是对需要补充的地方的说明。


❞

组件数据


需包含以下3个配置



  • eventEnums: 组件暴露出的事件枚举
  • ELREF: 如果组件使用到了element组件需将该组件Ref暴露出来，存在多个时返回对象形式。未使用则为null
  • style: 为组件样式相关配置

组件数据描述


❝

为了对组件数据更好的分类和描述，需创建interface.ts来进行描述，下面为三个大的描述分类


❞


  • DynamicDataModal：为动态数据的分类（主要为组件接收的外部数据）
  • StaticDataModal：为静态数据分类（主要组件本身创建的数据项和一些描述数据）
  • Style: 为组件样式类型的描述

示例:


interface DynamicDataModal {
  buttonText: string// 按钮文字
}
interface StaticDataModal {
  eventEnums: EventItem[], // 事件枚举
  ELREF: object // EL原生接口
}
interface Style {
  padding: string, // 内间距
  backgroundColor: string, // 背景颜色
  width: string, // 宽度
  height: string, // 高度
  borderRadius: string, // 边框圆角
  borderColor: string, // 边框颜色
  borderWidth: string, // 边框宽度
  borderStyle: string, // 边框类型
  fontSize: string, // 字体大小
  fontFamily: string, // 字体
  fontWeight: string, // 字体加粗
  color: string, // 字体颜色
  name: string, // 图标class
}
interface EventItem {
  name: string, // 事件名称
  label: string, // 事件描述
  params?: Array<EventParams>, // 事件回调参数
  returnVal?: any // 事件返回值
}
interface EventParams {
  paramName: string, // 参数名称
  paramType: any, // 参数类型
  remark: string // 参数描述
}

组件配置项描述


❝

需要对需求中组件可配置样式进行描述，配置在config.js中


❞

attr: 为配置项描述数组（其中字段主要为如下）



  • name: 配置项名称
  • type: 配置项类型

❝

select: 下拉选项（需配合value字段使用）
input: 输入框
switch: 开关
color: 颜色
number: 计数器
…


❞


  • attr: 对应的属性字段
  • value: 配置项数据（如下拉数据）
