重绘与回流

重绘与回流

六月 02, 2023
  • JavaScript
❝

重绘与回流是我们很熟悉的两个词语，我们都知道重绘的开销低于回流，但对于其原因可能并不是那么清楚。下面就是对重绘和回流在渲染过程中的区别进行一些分享。


❞

回流


触发条件



  • 当DOM结构的修改引发DOM几何尺寸变化的时候，会发生回流（操作如下）。


  1. DOM元素几何属性变化，如width、height、padding、margin、left、border等。
  2. DOM节点发生增减或移动。
  3. 读写offset、scroll属性时，浏览器为了获取这些值，需要进行回流操作。
  4. 调用window.getComputedStyle。

回流过程



  • 触发回流时，会将渲染流水线完整的走一遍（主要包含以下流程）

❝

生成DOM树 => 样式计算 => 生成布局树 => 建立图层树 => 生成绘制列表 （前面的这些步骤为渲染主线程中进行）
=> 分图块 => 生成位图 => 浏览器进程 => 显卡缓存 => 显示器显示 （渲染主线程之外）


❞

重绘


触发条件



  • 当DOM的修改导致了样式的变化，并且没有影响几何属性的时候就会发生重绘

重绘过程



  • 由于没有DOM几何属性的变化，因此元素的位置信息不需要更新（流程如下）

❝

样式计算 => 生成绘制列表


❞

跳过了生成布局树和建立图层树，直接生成绘制列表，然后继续进行分块。生成位图等后面的操作。我们可以看到重绘的流程时包含在回流的流程中的，因此重绘不一定导致回流，但回流一定发生重绘。


合成



  • 在渲染的过程中还有一种情况是直接合成，如transform、opacity、filter这些属性可以实现合成也就是我们说的GPU加速。

合成加速的原因



  • 在合成的情况下，会直接跳过布局和绘制的流程，直接进入非主线程处理的部分，即直接交给合成线程处理，优势如下：


  1. 能够充分发挥GPU的优势。合成线程生成位图的过程中会调用线程池，并在其中使用GPU进行加速生成，而GPU 是擅长处理位图数据的。
  2. 没有占用主线程的资源，即使主线程卡住了，效果依然能够流畅地展示。
